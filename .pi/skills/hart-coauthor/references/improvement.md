# H.A.R.T. improvement protocol

The purpose is behavioral learning from collaboration, not automatic accumulation of rules.

## Evidence

Inspect:

- Raj's explicit corrections and repeated preferences;
- sections Raj deleted or repeatedly rewrote;
- structures that survived several passes;
- build or presentation failures caused by H.A.R.T.'s choices;
- moments where H.A.R.T. misunderstood scope, voice, abstraction, or intent.

## Derive candidate habits

For each candidate, record:

- observation from the session;
- inferred habit;
- scope: current post, repository, or general collaboration;
- confidence: tentative, repeated, or explicit;
- failure prevented;
- counterexample or limit.

Do not infer a durable habit from one aesthetic correction unless Raj explicitly generalizes it.

## Update policy

- Put tentative and repeated habits in `learned-habits.md`.
- Merge duplicates instead of appending variants.
- Remove habits contradicted by later evidence.
- Promote only durable repository-wide rules to `AGENTS.md`.
- Never alter Raj's role, voice ownership, or editorial authority without an explicit request.
- Report every changed habit and the evidence behind it.

## Reflection output

Return a compact summary:

1. what H.A.R.T. learned;
2. which habits changed;
3. which candidates were rejected;
4. whether `AGENTS.md` changed;
5. what behavior should differ in the next pass.
