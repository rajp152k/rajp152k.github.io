# Learned collaboration habits

This file records revisable habits derived from sessions. `AGENTS.md` remains authoritative.

## Active

### Preserve editorial subtraction

- **Evidence:** Raj repeatedly removed glossary, entity, relationship, and ontology projections after seeing them in context.
- **Habit:** Treat deletion as a structural decision. Do not restore removed material in later coherence passes.
- **Scope:** Repository.
- **Confidence:** Repeated.

### Formalize relations at the correct level

- **Evidence:** Raj identified `Transformation` as an awkward ontology node; the graph became clearer when transformation became an edge.
- **Habit:** Model a process as an edge unless it needs identity, properties, state, or provenance.
- **Scope:** Diagrams and ontologies.
- **Confidence:** Repeated principle from an explicit correction.

### Prefer focused projections

- **Evidence:** Four ontology diagrams were reduced to one coherent graph.
- **Habit:** Start with one diagram per conceptual section. Add another only when it answers a distinct question required by the reader path.
- **Scope:** Posts.
- **Confidence:** Explicit.

### Let implementation pressure test concepts

- **Evidence:** About Us becoming a page exposed the need for first-class pages and configured navigation in the SSG.
- **Habit:** Treat presentation requests as tests of the content model; update the model when the semantic distinction is durable.
- **Scope:** Blog and SSG collaboration.
- **Confidence:** Repeated.
